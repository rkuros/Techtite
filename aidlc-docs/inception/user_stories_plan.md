# Techtite ユーザーストーリー作成計画

## プロジェクト概要

Techtite: AIエージェント向けエディタ + ナレッジベース（Obsidianライクなシステム）

### 仕様書から読み取った主要要件

- フォルダでプロジェクト管理
- Obsidianのようなマークダウンドキュメントエディタ
- Code Repositoryも兼ねる
- 作業日記 → ブログ自動公開パイプライン
- コード ↔ ドキュメント ↔ ナレッジ ↔ ブログ ↔ SNS の自動連動
- ローカルファースト保存（S3/Google Driveへの自動同期オプション）
- 複数端末からの閲覧・編集（ローカル同期）
- **編集はAIがメイン、人間の編集はサブ**

---

## 確定事項（Q&A結果）

| # | 項目 | 決定内容 |
|---|------|----------|
| 1 | プラットフォーム | デスクトップアプリ優先 |
| 2 | AIエージェント | Claude Code |
| 3 | Code Repository | Git統合（コードとドキュメントの分離問題を解決） |
| 4 | ブログ公開先 | Zenn もしくは Note |
| 5 | SNS | X か Threads |
| 6 | クラウド同期 | S3優先（Google Driveは可能なら） |
| 7 | スコープ | すべての機能 |
| 8 | アクター | 個人開発者（レビュー・管理者） + AIエージェント（Claude Code複数並列） |
| 9 | ユースケース | AI Coding Agent主導で開発 → ドキュメントをObsidian形式で保存・確認・編集 → 過程を日記化 → ブログ公開（個人開発用途） |
| 10 | 人間の役割 | マークダウン編集（Obsidian準拠）、コードは閲覧・レビューのみ |
| 11 | AIエージェント | 同一プロジェクト内で複数並列稼働。ターミナル + ダッシュボードで管理 |
